Problem Statement 6: AI for the Indian Investor

1. Executive Summary & Problem Context

India houses 14 crore+ active demat accounts, yet the majority of retail participants operate blindly. They rely on "gut feeling," chase informal tips, miss pivotal regulatory filings, and lack the expertise to decipher complex technical chart patterns or mutual fund overlaps.

The Solution: PAISA provides the missing intelligence layer. It transforms complex, fragmented financial data into actionable, money-making decisions through a multi-agent AI architecture.

We directly addressed the core demands of Problem Statement 6 by engineering:

  • Opportunity Radar: A live scanner replacing basic alerts with deep signal-finding logic (Bulk Deals, Insider Trades).
  • Chart Pattern Intelligence (Predictor): An ML algorithm producing math-backed price targets and technical breakdowns natively within the UI.
  • Market Chat - Next Gen: A deeply integrated financial assistant powered by Gemini 2.0 Flash that reads the user's specific CAS / Portfolio Data.

2. System Architecture

PAISA achieves its autonomy, speed, and intelligence via a Master-Agent Orchestrator. A central Python FastAPI router delegates tasks to 6 distinct AI models based on the required context length, reasoning depth, and latency tolerance.

UI / Frontend Layer (React + Vite)

User-facing React application. Maintains real-time WebSockets/SSE connections for streaming AI chat and polling live stock APIs. Houses the Recharts visualizer for portfolio allocation X-Rays.

Router & Live Data Integration (FastAPI + yfinance)

The orchestration loop. Pulls real-time NSE data, volumes, and corporate disclosures via `yfinance` & `BeautifulSoup`. Structures prompt contexts.

Agent 1: The Predictor & Technical Oracle (scikit-learn)

Machine Learning layer running pure mathematical Linear Regressions on 2-year historical data. Calculates RSI, MACD crossovers, and SMA support/resistance independently of LLM hallucination risks.

Agent 2 & 3: Market Chat & Portfolio Analyst (Gemini 2.x Family)

Utilizes Google Gemini 2.0 Flash for lightning-fast conversation, and Gemini 2.5 Pro for deep portfolio gap analysis, XIRR calculations, and expense drag reviews.

Agent 4: The Signal Finder (Groq - Llama 3.3 70B)

Monitors text strings from bulk deals and management commentary. Groq’s high LPU speed enables near-instantaneous Natural Language understanding to flag high-probability 'Insider Buying'.

Agent 5 & 6: The Fallback & Sentiment Engine (Ollama + DeepSeek)

DeepSeek R1 ranks macroeconomic sentiment. If cloud APIs fail, the system falls back seamlessly to a local `Ollama Mistral:latest` instance—guaranteeing 100% uptime with zero API costs.

Communication & Error Handling Logic

  • Communication: The React frontend speaks to the Python backend via secure REST API endpoints (`/api/predict`, `/api/signals`, `/api/health`) and Server-Sent Events (SSE) for streaming the Market Chat responses token-by-token.
  • Error Handling (Multi-Tier Fallback): If the primary LLM (Gemini) encounters a rate limit (429) or failure (500), the orchestrator intercepts the error and routes the exact payload to Groq (Tier 2) → Ollama Local (Tier 3) → Hardcoded Rules (Tier 4). The UI never crashes; the user always receives an actionable answer.

3. Tool Workflows (As per Problem Statement 6)

1. Opportunity Radar (Signal Finder)

Unlike standard screeners that just output raw data, the Opportunity Radar continuously scrapes Nifty 50 stocks for specific market anomalies (e.g., volume spikes > 2x average, institutional bulk deals). It feeds this raw text to an LLM, which calculates a "Confidence Score" and categorizes it as BULLISH or BEARISH. (Replaces news reading with autonomous opportunity hunting).

2. Chart Pattern Intelligence (Stock Predictor)

The user inputs an NSE ticker (e.g., RELIANCE). In 3 seconds:

  • Data Fetch: Pulls real-time quote and 2 years of daily historical data.
  • Pattern Recognition: Calculates Support (Floor) and Resistance (Ceiling) levels based on moving averages and prints them.
  • Prediction: A local ML regression generates a 7-day and 30-day percentage target path.
  • Explainability: A fast, rule-based AI algorithm evaluates the RSI (is it overbought?) and MACD momentum, generating a plain-English explanation of why the chart looks the way it does.

3. Market ChatGPT — Context Aware

Most AI agents hallucinate financial responses because they lack context. PAISA injects the user's uploaded portfolio (simulated via global context) directly into the system prompt. When a user asks "Am I exposed to the IT Sector drop?", the agent reads their specific overlapping Mutual Funds and provides a tailored rebalancing recommendation.
